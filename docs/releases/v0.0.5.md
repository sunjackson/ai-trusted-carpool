# 可信拼车 v0.0.5 / Trusted Carpool v0.0.5

## 中文

- Windows 与 Linux AppImage 现在使用 Tauri 官方签名更新协议：先检查、显示下载进度，再由用户明确选择安装并重启。
- 发车或上车仍在进行时只允许下载；后端会拒绝安装，结束拼车后可继续使用已经验证的更新包。
- Windows NSIS 在发布 CI 中使用固定证书指纹签名并执行 `signtool verify`；Windows/Linux 更新产物还会用应用内置公钥再次验签，并发布 SHA-256 清单。
- macOS 因尚无 Developer ID 与公证，DEB 因包管理器边界，继续从官方 Release 页面手动安装，不会出现在自动更新清单中。
- 签名、下载或安装失败不会替换当前版本；界面保留固定官方发布页作为安全回退。
- 客户端还会校验签名内文件名、下载地址与声明版本，避免旧的合法签名安装包被更新元数据伪装成更高版本。

安装和使用前请继续阅读仓库中的 `LEGAL.md`。本版本不增加遥测，也不会把账号凭据、会话密钥或请求正文发送到更新服务。

## English

- Windows and Linux AppImage builds now use Tauri's signed updater protocol with an explicit check, download progress, and user-confirmed install-and-restart flow.
- An active hosted or joined ride may download an update, but the Rust backend refuses installation until the ride ends; the verified download remains pending.
- Release CI signs Windows NSIS artifacts with a pinned certificate fingerprint and verifies them with `signtool`; Windows/Linux updater artifacts are also verified against the public key embedded in the app and ship with SHA-256 checksums.
- macOS remains a manual GitHub Release install until Developer ID signing and notarization are available. DEB packages also remain manual to preserve package-manager ownership. Neither is included in `latest.json`.
- Signature, download, or installation failures leave the currently installed version untouched, with the pinned official Releases page available as a fallback.
- The client also binds the signed filename to the download URL and advertised version so older legitimately signed installers cannot be replayed as a newer release.

Please review `LEGAL.md` before installing or using the application. This release adds no telemetry and never sends account credentials, session secrets, prompts, or responses to the update service.
