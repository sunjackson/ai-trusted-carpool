## 中文

- 新增“全天发车”：只要车主电脑和应用保持在线，车辆即可持续开放。客户端每 60 秒续签 3 分钟协调租约，关机、退出或断网后会自然离线。
- 车主重启应用后可选择恢复原通道或开新车。恢复会保留原车队编号与四个座位码，但不会恢复旧会话授权，乘客必须重新认领。
- 首页新增发车记录和上车记录，显示在线、等待开放、离线、已过期和已结束状态；仍在线的车辆支持一键重新上车。
- 新增私有 `ride-history.json`，使用原子写入、损坏隔离、Unix `0600` 权限和 100 条上限。它只保存恢复所需车队摘要与座位码，不保存账号凭据、设备私钥、access/session secret、信令、提示词或响应正文。
- 固定时段车辆同样使用短协调租约，同时继续执行独立业务结束时间，避免车主离线后邀请长期可发现。

安装和使用前请阅读仓库中的 `LEGAL.md`。本测试版没有 Authenticode、Developer ID 或 Tauri 自动更新签名；请只从本项目 GitHub Releases 下载并校验 SHA-256。

## English

> This is an unsigned test prerelease for opt-in downloads, testing, and self-hosting only. It is excluded from in-app updates. Windows SmartScreen and macOS Gatekeeper may show unknown-publisher or notarization warnings; verify the download against this Release's `SHA256SUMS.txt` before installing.

- Adds all-day hosting: a car remains available while the host computer and app stay online. The client renews a three-minute coordinator lease every 60 seconds, so shutdowns, exits, and network loss naturally take it offline.
- After restarting, a host can restore the original channel or start a new car. Restore preserves the car ID and four seat codes, but never restores old session authorization; passengers must claim again.
- Adds separate hosting and passenger history with online, scheduled, offline, expired, and stopped states. A passenger can rejoin only while the car is available.
- Adds a private `ride-history.json` with atomic writes, corrupt-file quarantine, Unix `0600` permissions, and a 100-record limit. It stores only the car summary and seat codes needed for recovery, never account credentials, device private keys, access/session secrets, signaling, prompts, or response bodies.
- Fixed-window cars now use the same short coordinator leases while independently enforcing their configured business end time.

Read `LEGAL.md` before installing or using the application. This test build has no Authenticode, Developer ID, or Tauri updater signature; download it only from this project's GitHub Releases and verify its SHA-256 digest.
