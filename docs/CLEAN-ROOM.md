# Clean-room 行为融合决策记录

## 目的

可信拼车可以参考其他桌面工具公开呈现的用户需求与交互结果，但实现必须保持 Apache-2.0 项目的独立来源。本记录用于约束当前路线图及后续修改者，避免把行为参考误变成代码或内容复制。

## 参考边界

- Cockpit Tools 以 CC BY-NC-SA 4.0 发布；本项目不导入、翻译、改写或运行其源代码，也不复制其目录结构、状态模型、界面布局、文案、图标或其他资产。
- 可参考的只有公开可观察的产品问题，例如“区分多个客户端实例”“先预览再导入”“导出脱敏诊断”“签名验证更新”。这些问题由本项目的威胁模型、现有接口和测试重新定义。
- 设计与实现只使用本仓库原有代码、平台/依赖官方文档以及独立编写的测试。若未来需要复用第三方实现，必须先完成许可证评估，并在独立提交中明确归属；不得混入本 clean-room 路线。

## 独立设计决策

| 需求 | 本项目独立约束 | 实现证据 |
| --- | --- | --- |
| 客户端实例管理 | 键为 `(accessId, tool)`；注册表不得保存会话密钥；关闭前必须同时校验 PID、启动时间、可执行文件指纹和拼车 profile | `client_launcher.rs`、`client_process.rs` 及进程复用/多车测试 |
| 账号健康 | 只持久化枚举化原因与时间，不保存上游正文或凭据；OAuth 刷新权仍归官方客户端 | `account_router.rs`、账号摘要与手动重试测试 |
| 持久化诊断 | 写入前统一脱敏；日志限时限量；支持包采用明确允许清单并对账号标识哈希化 | `diagnostics.rs`、`debugLog.ts` 及泄漏扫描测试 |
| 导入与恢复 | 凭据留在 Rust；预览会话短时、一次性；提交必须原子化 | v0.0.4 的接口和失败不变性测试 |
| 加密备份 | 使用现有密码学依赖独立定义格式；随机 salt/nonce；认证失败不得触碰账号池 | v0.0.4 的格式说明与篡改/口令测试 |
| 自动更新 | 只安装 Tauri 签名产物；活跃拼车期间禁止安装重启；各平台发布能力分别启用 | v0.0.5 的更新配置、CI 验证和平台门禁 |

## 禁止事项

1. 不以“兼容 Cockpit”为由接受其私有字段、备份格式、内部命名或页面结构。
2. 不把外部项目的截图、文案或资源作为实现输入。
3. 不在日志、诊断包、更新清单或实例注册表中扩大凭据和会话数据边界。
4. 后续提交若改变上述边界，必须更新本记录、增加回归测试并重新做许可证与安全审查。
