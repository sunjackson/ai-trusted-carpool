# 可信拼车桌面端产品基线

## 产品边界

只做两件事：**一键发车**与**一键上车**。产品前期面向彼此认识、彼此信任的人，不引入押金、积分、结算、扣罚或用户等级。账号检测、官方地址校验、设备绑定、网络连接和四座并发全部在后台自动完成。

## 普通用户主流程

- 发车：选择已检测到的本机官方 Claude Code 或 Codex 账号，设置固定开始—结束时间或“全天发车”，点击“开始发车”，复制四个上车码。
- 上车：粘贴上车码，选择 Claude Code 或 Codex，点击“上车并打开”。
- 全天车只在车主设备在线时开放。车主重启后可以恢复原车队编号和四个上车码，也可以开新车；恢复不会恢复旧会话授权。
- 首页保留车主发车记录与乘客上车记录；乘客记录显示在线、等待开放、离线、过期状态，仅在线车辆允许重新上车。
- 一辆车最多四名乘客并发；同一用户可加入多辆车；每辆车只有一个发起者。
- 车主实时查看每个座位、每个模型的请求次数、输入 Token、输出 Token、缓存读取、缓存写入和最近使用时间。
- 成员列表只展示昵称、工具、总 Token、请求数、官价估算和关键限额；点击成员后再展示按模型明细，避免主页面堆满数据。
- 车主可为每名成员分别设置 5 小时、24 小时、7 天滚动 Token 限额。达到任一限额后停止新的官方请求；留空表示不限额。
- 车主与所有在线成员均可看到该车实际 Claude/Codex OAuth 账号的官方剩余额度和重置时间。API Key 没有订阅额度查询能力时显示“不可用”，不伪造数据。
- 客户端按请求发生时间累计“官方 API 标准价估算（USD）”；它不是实际账单，也不代表 Claude/Codex 订阅额度。未知模型显示“暂无官价”，绝不套用相近模型价格。

## 跨平台目标

统一支持 macOS、Windows 与 Linux。客户端使用 Tauri 2 + Rust + React/TypeScript；本地凭据不上传，车主只连接官方 API 地址；优先 WebRTC 直连，失败时使用受控 TURN 中继。

## 视觉原则

普通页不出现 P2P、WebRTC、TURN、密钥、会话、审计等术语。每个页面只有一个主要动作，状态使用人类语言表达，高级诊断折叠隐藏。

成员主列表采用渐进披露：重要信息一眼看完；按工具/模型拆分、缓存分类和限额编辑只在成员详情中出现。乘客只能看到自己的明细，但所有成员看到相同的车队账号额度。

## 价格口径

官价目录只采用 OpenAI 与 Anthropic 官方公开价格，并记录核验日期。缓存读、缓存写及长上下文按官方不同费率分别计算；调价前后的请求按各自发生时间保留历史价格。详见 [`PRICING-SOURCES.md`](./PRICING-SOURCES.md)。
