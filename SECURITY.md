# 安全政策

可信拼车处理官方账号凭据的本机代理、端到端加密隧道与设备身份签名。我们非常重视任何可能影响这些边界的漏洞报告。

## 报告渠道

**请不要在公开 issue 中披露安全漏洞。**

- 首选：GitHub 私密漏洞报告（仓库 Security 标签页 → [Report a vulnerability](https://github.com/sunjackson/ai-trusted-carpool/security/advisories/new)）。
- 报告请尽量包含：影响版本、复现步骤或 PoC、影响评估，以及你认为可行的修复方向。

## 响应承诺

| 阶段 | 时限 |
| --- | --- |
| 确认收到报告 | 72 小时内 |
| 初步评估与严重度分级 | 7 天内 |
| 修复或缓解措施 | 视严重度，高危目标 30 天内 |
| 协调披露 | 修复发布后，或自报告起最迟 90 天 |

感谢负责任的披露。我们会在发布说明中致谢报告者（除非你希望匿名）。目前没有漏洞赏金计划。

## 覆盖范围

以下问题属于本政策范围：

- 桌面客户端：凭据泄露、加密实现缺陷（`src-tauri/src/crypto.rs`、`identity.rs`）、本地代理越权（`local_proxy.rs`）、中继校验绕过（`relay.rs`，例如把请求转发到非官方地址）、深链/上车码解析绕过（`join_link.rs`）、桌面客户端配置备份恢复缺陷（`client_launcher.rs`）。
- 协调服务参考实现（`deploy/coordinator/`）：签名校验绕过、限速绕过、邀请或信令伪造。
- 构建与发布链路：CI 工作流投毒、安装包完整性。

以下不属于本政策范围：

- 「多人共享官方账号违反服务条款」本身——这是已在 [LEGAL.md](LEGAL.md) 中声明的产品边界，不是漏洞。
- 需要车主本机已被完全攻陷才能实现的攻击。
- 针对 Anthropic、OpenAI 官方服务的漏洞（请报告给对应厂商）。

## 支持版本

| 版本 | 是否接收安全修复 |
| --- | --- |
| 最新发布版（`main` / 最新 tag） | 是 |
| 更早版本 | 否，请升级 |
