# 更新日志

本文件记录项目的所有重要变更，格式遵循 [Keep a Changelog](https://keepachangelog.com/zh-CN/1.1.0/)，版本号遵循 [语义化版本](https://semver.org/lang/zh-CN/)。

## [Unreleased]

## [0.0.6] - 2026-07-22

### Added

- 新增“全天发车”：车主保持电脑与应用在线即可持续开放，协调服务使用 3 分钟短租约并每 60 秒续签，关机或断网后自动离线。
- 新增发车与上车历史。首页展示在线、等待开放、离线、过期和已结束状态；仍在线的乘客记录可直接重新上车。
- 新增车主重启恢复流程：可恢复原车队编号与四个座位码，或归档旧通道后开新车。

### Changed

- 固定时段车辆也改用短协调租约，同时继续独立校验业务结束时间，避免协调记录在车主离线后长期可发现。
- 恢复通道时只恢复车队身份与座位码，不恢复旧 access id、session secret、设备绑定、信令队列或 WebRTC 连接；乘客必须重新认领。

### Security

- 新增私有 `ride-history.json`，采用原子替换、损坏隔离、Unix `0600` 权限和最多 100 条限制；只保存恢复所需车队摘要与座位码，不保存账号凭据、设备私钥、会话密钥、提示词或响应正文。
- 本版本以 `v0.0.6-test.1` 未签名测试预发布版提供手动下载，不生成自动更新签名或 `latest.json`。

## [0.0.5] - 2026-07-22

### Added

- 新增 Tauri 官方签名更新流程：启动时检查、显示下载进度、保留已验证更新，并由用户显式选择安装与重启。
- 发布流水线新增独立更新签名、内置公钥加密验签、严格的 `latest.json` 生成、全部产物 SHA-256 清单，以及中英双语 Release Notes。

### Changed

- Windows NSIS 正式 tag 构建改为导入 PFX、固定证书指纹并执行 `signtool verify`；普通分支构建继续明确标记为未签名开发产物。
- Linux AppImage 支持应用内更新；DEB 继续由包管理器或官方 Release 页面手动安装。macOS 自动安装代码保留在显式开关后，取得 Developer ID 并完成公证演练前不进入更新清单。
- macOS 通用构建脚本从应用配置读取版本号，不再硬编码 DMG 文件名。

### Security

- 应用只安装内置公钥验证通过的更新包；签名、下载或安装失败时保留当前版本与已验证待安装状态。手动 Release 页面提醒使用独立文案，不会冒充应用内已验签更新。
- 客户端会把 Minisign 已认证文件名与下载 URL、平台、CPU 架构及完整 SemVer 绑定，阻止跨平台/架构交换，以及把 prerelease、build 或旧签名产物伪装成稳定高版本回放。
- 发车、上车及其启动过渡期间仅允许下载更新，Rust 后端在安装前再次强制检查，并在安装调用期间阻止新的拼车启动。
- 更新产物会在构建端与发布端分别用内置公钥验证内容签名和签名内文件名；清单生成对平台、架构、版本与 tag 进行 fail-closed 校验，且明确排除 macOS 与 DEB 自动安装目标。Release 先以 draft 上传并核对全部远端资产，再原子公开。

## [0.0.4] - 2026-07-22

### Added

- 新增 10 分钟、一次性的 Rust 账号导入预览会话；本机配置、粘贴内容和多文件批次仅向前端返回脱敏的账号类型、名称、来源及新增/更新/冲突状态，确认后一次性原子提交。
- 账号存储升级到 v2，增加稳定来源标识；首次读取 v1 时保留已认证的 `accounts.json.v1-authenticated.bak`，再原子迁移。
- 新增 `.tcarpool-backup` 加密账号备份，以及合并/替换恢复预览；替换恢复需要双重界面确认和后端显式确认，并在写入前创建可自动回滚的本机快照。

### Changed

- OAuth 凭据轮换依次按官方账号标识、本机来源槽位和稳定凭据身份匹配，更新凭据时保留用户设置的名称、优先级和启用状态。
- 旧 `import_accounts` 与 `import_local_accounts` 接口保留一个版本，但内部统一转接新的预览与原子提交路径。

### Security

- 备份使用 PBKDF2-HMAC-SHA256（210,000 次）派生密钥、AES-256-GCM 认证加密及随机 salt/nonce；文件限制为 24 MiB，错误口令、篡改、未知版本和截断内容均不得修改现有账号池。
- 备份只包含账号凭据、账号元数据、优先级和稳定来源标识；默认排除日志、设备身份、邀请、会话密钥、客户端 profile、运行态和协调服务消息。

## [0.0.3] - 2026-07-22

### Added

- 桌面客户端按“车辆授权 + 工具”登记受管实例；新增实例列表、精确定位和精确关闭接口，同一用户可同时加入多辆车而互不覆盖。
- 账号路由健康状态独立持久化，记录枚举化失败原因、指数冷却、最近成功/失败和同优先级 LRU；账号管理页显示脱敏状态并支持单账号立即重试。
- 启动、发车前和运行期间定期检查官方客户端凭据，只轮换曾经导入的账号；发现新账号仅提示，不自动加入共享池。
- Rust 与前端统一写入脱敏 JSONL 诊断日志，内存保留 500 条，磁盘默认保留 7 天且总量不超过 20 MB；三击 Logo 面板新增搜索、级别/来源筛选、历史日志、清空、日志目录和诊断包导出。

### Changed

- 客户端启动最多等待 8 秒，仅在匹配正确可执行文件和独立拼车 profile 的真实进程就绪后返回；失败自动回滚，重复点击同一实例改为定位已有窗口。
- 离车、退出和崩溃恢复只按 PID、启动时间、可执行文件指纹及 profile 关闭对应受管实例；Microsoft Store 客户端无法隔离时拒绝与普通客户端并行，不再强制关闭。
- 旧版单工具路由首次启动只恢复配置备份并清理旧 profile，不再根据应用名猜测并终止进程。

### Security

- 诊断写入前统一清理 API Key、OAuth/JWT、Authorization、Cookie、密码、会话密钥、上车码、邮箱、用户目录及请求正文；诊断包只包含脱敏日志、版本/系统信息、客户端检测和哈希化路由健康摘要。
- 本阶段依据外部产品的公开行为进行独立设计，不复制其代码、结构、文案或资产；决策边界记录于 `docs/CLEAN-ROOM.md`。

## [0.0.2] - 2026-07-22

### Added

- 新增加密本机账号池、批量/本机凭据导入、账号优先级与最多三候选的安全重试路由、官方额度查询和账号管理界面。

### Fixed

- 同网（同公网 IP）上车时乘客卡在「连接中」超时：协调器 poll 改为优先按 peer 限速并提高同 IP 配额，车主认领轮询从 400ms 放慢到 1s，避免挤占乘客的 answer/ICE 信令。
- 车主座位在授权后先显示「连接中」，WebRTC 数据通道真正打开后再变为「已连接」；提前到达的 ICE 候选在车主侧短时缓冲。

## [0.0.1] - 2026-07-21

### Added

- 乘客 0 门槛一键启动：缺少 Claude Code / Codex 命令行时，应用直接从官方渠道下载原生独立程序（Claude 走 `downloads.claude.ai` 官方清单流程，Codex 走 GitHub `openai/codex` 官方发布），SHA-256 强校验后原子启用到应用托管目录，无需 Node.js、无需管理员权限；乘客选择工具页提供"下载并打开"单动作，实时字节级进度、可取消。
- 应用托管运行时管理：`current` 指针原子切换、`provenance.json` 来源记录（含下载与二进制双哈希）、后台缓存官方最新版本号（24 小时）并提供一键更新；托管的 Claude 禁用其自更新，由本应用统一管理；用户自装版本始终优先。
- 升级安全：安装/更新总是优先拉取最新官方元数据（缓存仅作离线兜底）；新二进制必须通过 `--version` 冒烟自检才切换 `current` 指针，失败自动保留旧版本；保留最近一个旧版本作为回滚路径，更早版本自动清理。
- 启动完整性门禁：每次启动应用托管的 CLI 前重新计算 SHA-256 并与安装时记录比对，不一致立即拒绝启动、清除 `current` 指针并回到一键安装状态；托管目录权限收紧为仅当前用户（Unix 0700）。
- 应用自身更新提醒：对比 GitHub 官方最新 Release（缓存 24 小时），标题栏出现"新版本 vX.Y.Z"提示，点击打开固定官方发布页（不信任接口返回的任意 URL）。
- P2P 信令端到端加密：SDP/ICE 候选/离开通知在送达协调服务前用 X25519 + AES-256-GCM 信封加密（与授权信封域隔离），协调服务全程看不到会话描述、候选 IP、座位码与授权编号。
- P2P 稳定性：乘客断线自动恢复（ICE 失败/通道关闭/心跳超时触发，指数退避 6 次快速重试后转 30 秒持续重试，重连前自动刷新 TURN 时效凭据）；车主把已绑定乘客的新 offer 视为替代旧会话，崩溃后可立即重新上车；使用中页面实时显示连接状态（已连接/重连中/中断）。
- P2P 健壮性：乘客信令轮询逐条校验解密、跳过非法消息，杜绝"一条垃圾消息导致同批合法信令永久丢失"（协调服务信箱取走即删）；车主待处理信令队列加 256 条上限。
- 信令轮询改为自调度循环并带失败指数退避（700ms 起、5 秒封顶，成功即恢复），协调服务故障期间不再全速空转；车主侧连接销毁时同步清理在途请求映射。
- WebRTC E2E 上 CI：新增每周定时 + 手动触发的 workflow（Ubuntu 自带 Chrome，真实官方 TURN），测试脚本的 TURN 地址改为动态 DNS 解析并识别代理假 IP（198.18/15）自动回退，兼容 Linux Chrome/Chromium 路径。
- npm 一键安装保留为兜底（参考 CC Switch）：官方直连不可达且本机有 Node.js 时自动回退，权限/网络失败给出可读提示与手动命令。
- 工具检测增强：显示 CLI 版本号（托管版本直接读取，npm 安装读取全局包 `package.json`，不拖慢检测）。
- 开源基础设施：Apache-2.0 许可证、NOTICE、贡献指南、行为准则、安全政策、issue/PR 模板与 Dependabot 配置。
- 使用须知与免责声明（`LEGAL.md`），应用首次启动时的一次性风险确认页。
- Open Core 商业模式说明（`docs/BUSINESS-MODEL.md`）、自建部署指南（`docs/SELF-HOSTING.md`）与发布签名清单（`docs/RELEASE.md`）。
- 协调服务参考实现新增 `/api/v1/turn-credentials` 接口（coturn REST API 时效凭据方案），自建 TURN 中继可用；凭据发放改为**签名 POST**（证明持有设备私钥），拒绝匿名 `GET ?peer_id=` 刷中继。
- 公开协调器滥用防护：邀请注册、发信、轮询、TURN 按 IP/peer 限速；每个车主身份最多 16 条活跃邀请；免网站登录即可用，控成本靠配额而非账号墙。
- 客户端的 TURN 域名校验与官方上车链接域名改为从已配置的协调服务地址派生，自建部署无需改代码（默认仍为 `p2p.cnaigc.ai`）。
- 前端最小 i18n 骨架（默认 `zh-CN`，可选 `en`），首启确认页与欢迎页已迁移。
- CI 安全加固：`cargo audit` 与 CodeQL（JavaScript/TypeScript + Rust）扫描；Vitest 覆盖率报告。

### 初始开发（2026-07）

- Claude Code 与 Codex 的一键发车/一键上车核心流程：签名邀请、设备绑定、四座并发。
- WebRTC 直连优先、TURN 兜底的端到端加密转发，本地 HTTP 代理流式透传官方响应。
- 按成员 → 工具 → 模型的实时用量统计与官方 USD 标准价估算；成员 5 小时/24 小时/7 天滚动 Token 限额。
- 车主官方 Claude/Codex OAuth 账号剩余额度同步展示；API Key 无额度接口时明确显示不可用。
- 官方桌面客户端安全接入（配置写入前完整备份、离车/退出/下次启动时恢复）。
- macOS 菜单栏、Windows 托盘与 Linux 状态区常驻状态；关闭主窗口后驻留后台。
- 本地追加式用量历史（不含提示词、响应、密钥）；一键上车官方链接与深链解析的严格校验。
- GitHub Actions 多平台安装包构建（macOS 通用 DMG、Windows NSIS、Linux DEB/AppImage）与打 tag 自动发布。
